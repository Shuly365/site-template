# **אפיון אתר "צ'אט-הכשרה"**
כדי לתקוף אתרים עליכם לדעת כיצד הם בנויים לכן נבנה אתר פשוט באמצעות הטכנולוגיות הנפוצות כיום תוך דגש על מימוש יכולות Web חשובות.
על מנת למנוע התעסקות מיותרת עם הקמת פרוייקט מ0, קיימת תבנית זו שמכילה Front יחסית מוכן! הוא מכיל את העמודים הבאים:

* **עמוד התחברות**
* **עמוד הרשמה**
* **עמוד שיחה**
* **עמוד אדמין**

עליכם להשתמש בתבנית על מנת לפתח את אתר צ'אט ההכשרה הכולל -  מספר חדרי שיחה (שאדמינים מסוגלים לייצר ולמחוק) ניהול צד שרת וצד לקוח, שימוש במסד נתונים, הזדהות, הרשאות, העברת מסרים מיידית והצפנה. *שימו לב שעליכם יהיה לתקן לוגיקה, אבטחה ופונקציונליות במידת הצורך!*
## הגדרת מוצר סופי - 
### התחברות/הרשמה
יבוצע באמצעות שם משתמש וסיסמא באמצעות הזדהות מול מסד נתונים.
*שימו לב - השתמשו ב-Hash.*

### עמוד בית - שיחה נוכחית + פאנל בחירת שיחה
עמוד שיחה בו כל המשתמשים יכולים לרשום הודעות ולראות את ההודעות ששלחו אחרים (כמו קבוצה בוואצאפ).
על ההודעות להתעדכן באופן אוטומטי ולא באמצעות refresh לדפדפן.
הודעות שנשלחות על ידי משתמש בהרשאת אדמין יהיו מוצגות עם כוכב בהודעה.
בעמוד השיחה ישנה דרך להחליף לשיחה אחרת באופן אינטואיטיבי. ואדמינים יוכלו להוסיף ולהסיר חדרי שיחות.

### פאנל אדמין
פאנל המוצג אך ורק למשתמשי אדמין בו ניתן לראות ולמחוק משתמשים במערכת. *משתמשים הנמחקים מהמערכת לא יוכלו לראות לקבל ולשלוח הודעות.*
בעמוד הבית יוצג כפתור מעבר לפאנל האדמין אך ורק למשתמשים בהרשאת אדמין.

### ניהול הרשאות
יש לממש ניהול הרשאות שבאמצעותו כל משתמש יוכל לגשת אך ורק לנתיבים ולמידע המורשים לו. ניהול ההרשאות יבוצע גם בצד הלקוח (הגנה על הצגת המידע) וגם בצד השרת (הגנה על שליפת המידע) באמצעות JWT. במידה ומשתמש מבצע פעולה שלא ברמת הרשאותיו, יש להציג הודעה מתאימה.

### נקודות חשובות!
* המערכת תפותח באמצעות: MongoDB, ExpressJS, React, Node. אם ברצונכם להשתמש בטכנולוגיה אחרת ל-Backend או ל-DB, **אתם מוזמנים!** בתנאי שהחופף מאשר (*מומלץ שתדעו לנמק את הסיבה לשימוש בכל טכנולוגיה*).
* את צד הלקוח יש להמשיך לממש בצורה רספונסיבית על מנת שיוצג נכון לכל רזולוציה.
* אימות קלט בצד הלקוח ובצד השרת.
* על האתר להשתמש ב-HTTPS
* לעשות Docker Image ל-Docker Desktop

מסמך זה נותן קווים מנחים. אך אם תראו לנכון להוסיף או לשנות דברים, התייעצו עם החופף.

עבודה נכונה - אפיינו את האתר מבחינת טכנולוגיות וארכיטקטורה ולאחר מכן קבלו את אישור החופף לפני תחילת העבודה.
